# Use Case: актуализация расписания мастеров

| Поле | Описание |
|---|---|
| Идентификатор | UC-mng-01 |
| Название | Актуализировать расписание мастеров |
| Цель, назначение | Актуализировать расписание мастеров |
| Основное действующее лицо | Роль: менеджер<br>Потребность: поддерживать актуальность расписания в целях предотвращения накладок и ошибок |
| Другие заинтересованные стороны | Мастер<br>Интерес: эффективно планировать свое рабочее время<br><br>Клиент<br>Интерес: осведомленность об актуальном расписании интересующих мастеров |
| Предусловие | Менеджер авторизован в системе<br>В систему загружен список мастеров<br>В систему загружено текущее расписание мастеров |
| Триггер | Поступает информация о том, что требуется изменение расписания мастера |
| Минимальные гарантии | Старое расписание сохранится в системе, а менеджер получит уведомление, что при сохранении произошла ошибка |
| Гарантии успеха | Обновленное расписание сохранится в системе |
